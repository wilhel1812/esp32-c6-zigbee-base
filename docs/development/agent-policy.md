# Agent Policy

Agents must follow the repository `AGENTS.md` file.

Important defaults:

- Reuse or adapt existing code before adding new code.
- Do not add UI elements without approval.
- Mention code or UI that looks removable or consolidatable.
- Bump version metadata for meaningful changes.
- Build the generic example before committing.
- Commit successful changes by default.
- Push after committing unless blocked or explicitly told not to.
- Do not create release tags unless explicitly requested.
